# Дипломный проект профессии «Python-разработчик: расширенный курс»

Cервис для автоматизации процессов закупок в розничных сетях.

---

## Технологический стек

* **Язык программирования:** Python 3.10+
* **Фреймворк:** Django 4.x
* **API:** Django REST Framework (DRF)
* **Аутентификация:** Token Authentication / Custom User Model
* **Тестирование:** Django Test Suite (`unittest`)
* **СУБД:** SQLite (по умолчанию, легко масштабируется на PostgreSQL)

---

## Основной функционал и реализованные API Endpoints

### 1. Пользователи и Авторизация
* **Регистрация пользователя** с подтверждением по Email (генерация токена подтверждения).
* **Авторизация / Логин** с выдачей токена доступа.
* **Управление профилем** и сброс пароля.

### 2. Каталог и Товары
* **Импорт прайс-листов:** Загрузка товаров и параметров от поставщиков/магазинов (JSON формат).
* **Просмотр каталога:** Получение списка доступных товаров, фильтрация по категориям и магазинам.
* **Спецификация товара:** Получение детальной информации о конкретной позиции (включая параметры, цены и остатки).

### 3. Корзина (`BasketView`)
* `GET /api/v1/basket/` — получение текущей корзины пользователя с детальной вложенной информацией о товарах.
* `POST /api/v1/basket/` — добавление товаров от разных магазинов в корзину.
* `PUT /api/v1/basket/` — изменение количества товаров в корзине.
* `DELETE /api/v1/basket/` — удаление позиций из корзины.

### 4. Контакты и Адреса доставки (`ContactView`)
* Просмотр, добавление, обновление и удаление адресов и контактных данных пользователя.

### 5. Заказы (`OrderView`)
* **Оформление заказа:** Перевод корзины в статус заказа с привязкой адреса доставки.
* **Уведомления:** Автоматическая отправка Email с подтверждением деталей заказа покупателю и уведомлением магазинов.
* **История заказов:** Получение списка всех оформленных заказов и их детализация.
* **Статусы заказов:** Изменение и отслеживание статуса обработки заказа.
